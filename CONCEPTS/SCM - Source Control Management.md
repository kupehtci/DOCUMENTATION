# SCM or Source Control Management

An SCM or Source Control Management is a system that is used to track, version and manage changes made to source code and other kind of files in software developent. 

The SCM maintains a history of the modifications made to the code helping with: 
* **Collaboration**: multiple developers can work on the same code, overwriting each other changes and managing the versioning of the project. 
* **History tracking**: the changes over the code are recorded with a detailed history, including author, changes made and other details together with reverting possibility. 
* **Branching and merging**: developers can create "branches" [[GIT - Change branch]] for implementing features and keep a separate track of changes and version, that can be merged back into the original branch when its ready. 
* **Backup and recovery**: allows to recovery, 


Its also known as version control.
