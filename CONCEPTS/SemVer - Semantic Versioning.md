# SemVer - Semantic Versioning

SemVer or Semantic Versioning is an standard scheme for assigning numerical numbers to software releases to communicate the versions, nature and impact of the change. 

The scheme follows the `X.Y.Z` or `MAJOR.MINOR.PATCH` format, where the number increments depending on the type or nature of the change: 
* MAJOR for high impact changes that make the software incompatible with previous versions. 
* MINOR for backward compatible inclusion of new features. 
* PATH for backward compatible bug fixes or software modifications that doesn't add new functionalities. 

The key rules in the semantic versioning are: 
* start the versioning at `1.0.0` for the first release that is stable or production ready. 
* Increment major only on breaking changes.
