# SonarQube - Dependency Check

OWASP Dependency Check ([[OWASP Dependency Check]]) is an SCA[^1] analysis tool that identifies the dependencies of the project and maps it with common known CVEs vulnerabilities. 

The report generated by OWASP contains CVEs identified in the dependencies of the code and this CVEs can be published into SonarQube as **Security Hotspots** using **dependency-check-sonar-plugin** (https://github.com/dependency-check/dependency-check-sonar-plugin). 


For integrating this functionality, the OWASP Dependency Check ([OWASP Dependency Check](https://owasp.org/www-project-dependency-check/))([[OWASP Dependency Check]]) must be executed over the project to generate the corresponding reports in JSON, HTML or XML formats so this plugin can publish the results in SonarQube. 


The parameters that you need to define to enable the reports publishing are defined in [[SonarQube - CI-CD Parameters]]. 

[^1]: SCA or Software Composition Analysis  [[SCA Software Composition Analysis]]
