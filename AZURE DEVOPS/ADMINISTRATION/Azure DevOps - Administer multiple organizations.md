# Azure DevOps - Administer multiple organizations

Azure DevOps organizations provide a very useful management tooling to maintain an strong isolation between different projects that help the security, billing and of course the administration tasks over the repos. 
During several months, I have been in charge of managing multiple organizations, each one with its own different projects; isolated between them and the cost of standardizing a common DevOps workflow between then has been hard. 

In this guide I want to present some implementations that has help me a lot when managing this environment, so you have a little help if facing this same scenario.
